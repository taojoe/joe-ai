# Math Print Quiz Skill 指南

这个 Skill 用于帮助用户生成定制化的、A4 纸打印的数学知识讲解与小测验 PDF 文件。你需要作为一个教育辅助助手，引导用户生成高质量的内容。

## 🎯 工作流程 (Workflow)

请严格按照以下步骤执行此 Skill：

### Step 1: 需求讨论与主题确认
- **主动与用户沟通**：询问用户希望生成什么数学知识点的数据（例如：平方差公式、二次根式、一元二次方程等）。
- 确认要测试的范围以及是否需要特定类型的题目（如画图题）。该工具的主要目的是帮助学生**建立和完善数学概念**，并进行**基础练习**。
- **等待用户回复**，不要在没有明确主题的情况下生成数据。

### Step 2: 构思与生成 Markdown 数据文件
在确认知识点后，你需要帮用户构思内容并生成对应的 `.md` 数据文件（基于自定义的类似 MDX 格式）。**重要：必须使用带有时间戳或自增数字作为前缀的文件名**（存放在 `data/<日期时间或数字前缀>_<主题英文名>.md`），以确保即使相同的知识点多次生成，也不会覆盖旧文件。例如：`data/20260331_1430_sum-of-cubes.md`。

生成的数据必须严格遵守以下**内容生成标准**：

1. **整体篇幅约定**：输出必须适配 A4 纸排版，共计分为 **4 页**（1 页知识，2 页练习，1 页答案）。
2. **第一部分（第 1 页）：知识讲解**
   - 详细讲解该知识点的原理，**必须从多个维度（或多种推导方法）展开**。
   - 内容须极其充实，**必须完全撑起一整页的 A4 版面**，包含原理描述、多维推导过程、核心公式、避坑指南以及经典例题演示。
3. **第二部分（第 2、3 页）：练习题**
   - **题目数量要求**：生成 **4～6 道** 题目，这些题目会被平均分配到 2 页 A4 纸中，以确保有充足的留白作答。
   - **题目形式要求**：选择题 (multiple-choice)、简答题 (short-answer)、或画图题 (drawing)。
4. **第三部分（第 4 页）：答案与解析**
   - 为每道题提供详尽的答案和解题过程。

⚠️ **重大改变：告别 JSON 与烦人的转义规则！**
由于 JSON 中频繁的 `\\n` 与 LaTeX 后端渲染冲突，现在系统已升级为**直接支持基于纯 Markdown 文件**生成 PDF！你再也不需要手动打双斜杠转移 LaTeX。你可以自由、放心地像写普通 Markdown 一样书写 `\log`，`\frac`，甚至换行！

### Step 3: 运行生成命令
当你完成了 `*.md` 文件的生成或更新后，告诉用户即将运行脚本，并执行以下命令（注意文件名不含扩展名）：
```bash
bun run quiz -- <带前缀的md文件名(不含扩展名)>
# 例: bun run quiz -- 20260331_1430_difference-of-squares
```

## 📄 数据格式参考 (`data/*.md`)

你在 Step 2 中生成的文件必须符合以下结构（请直接将下面结构当做模板套用）：

```markdown
---
title: 知识点中文标题
subtitle: 英文副标题 (可选)
---

<Knowledge>
📖 知识要点
（在这里写大量且充实的 Markdown 格式知识说明。支持 Markdown 粗体，列表以及常规 LaTeX $...$。多维展开！）

$$
\text{单行公式可以这样居中展示: } a^2 - b^2 = (a-b)(a+b)
$$

### 核心要点
- 重点 1：必须...
- 重点 2：注意...
</Knowledge>

<Question type="multiple-choice" answer="A">
题目内容 (支持 LaTeX，例如求 $99 \times 101$)

<Options>
- $A. 选项 一$
- $B. 选项 二$
- $C. 选项 三$
- $D. 选项 四$
</Options>

<Explanation>
详细解题过程，因为 $(a-b)(a+b)$ ...
</Explanation>
</Question>

<Question type="short-answer" answer="答案写在这里">
简答题内容... 求 $\log_a N$

<Explanation>
详细解题过程...
</Explanation>
</Question>

<Question type="drawing" answer="本题为开放性画图">
画图题说明...

<Explanation>
画图的指导建议...
</Explanation>
</Question>
```
（注：题目总数请控制在 4 ~ 6 个，以保证合适的排版空间）
